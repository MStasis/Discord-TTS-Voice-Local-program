# Voiceboard 프로젝트 인수인계

이 문서는 다른 채팅이나 다른 작업자가 프로젝트를 바로 이어받을 수 있도록 현재까지 만든 기능, 코드 구조, 빌드/배포 상태, 주의점을 정리한 인수인계용 문서입니다.

## 1. 프로젝트 개요

- 프로젝트명: Voiceboard
- 저장소: https://github.com/MStasis/Discord-TTS-Voice-Local-program.git
- 로컬 경로: `C:\Users\Admin\Desktop\CODE\Discord_VoiceChat`
- 앱 형태: Electron 기반 Windows 데스크톱 앱
- 목적: Discord 봇 없이, 사용자 PC의 로컬 오디오 장치만 사용해서 TTS와 사운드보드를 Discord 음성 채널로 송출
- 대상 OS: Windows 10/11
- 가상 오디오 장치: VB-CABLE 권장

핵심 동작 방식:

1. Voiceboard가 TTS 또는 사운드 파일을 재생한다.
2. 앱 출력 장치를 `CABLE Input`으로 지정한다.
3. Windows 기본 녹음/통신 마이크를 `CABLE Output`으로 바꾼다.
4. Discord 입력 장치를 `Default`로 두면 Discord가 `CABLE Output`을 입력으로 사용한다.
5. 결과적으로 봇 없이 내 PC에서 재생한 소리가 Discord 음성 채널에 들어간다.

## 2. 현재 구현된 기능

### 기본 기능

- Discord 봇 없이 로컬 PC에서만 동작
- TTS 문장 입력 후 실행
- TTS 입력창에서 Enter로 즉시 전송
- 전송 성공 후 입력창 자동 비우기
- 자주 쓰는 문장 저장, 실행, 삭제
- 사운드보드 오디오 파일 가져오기, 실행, 삭제
- 사운드보드/YouTube 재생 중인 사운드 정지
- YouTube 링크 입력 후 오디오 재생
- YouTube 링크를 사운드보드 항목으로 저장
- 사운드보드 항목 편집으로 시작 위치와 길이를 지정해 MP3로 자르기
- 전송한 TTS 문장 로그 확인
- 로그에 있는 문장 클릭 또는 `다시 전송` 버튼으로 재전송
- 앱 데이터 폴더 열기
- 재생 중인 TTS/사운드 전체 정지

### TTS

- 기본 엔진: Microsoft Edge Neural TTS 계열 온라인 합성
- 기본 추천 음성: `ko-KR-SunHiNeural`
- 선택 가능한 음성:
  - `ko-KR-SunHiNeural`
  - `en-US-JennyNeural`
  - `en-US-AriaNeural`
  - `ja-JP-NanamiNeural`
  - `zh-CN-XiaoxiaoNeural`
- 속도, 피치, TTS 볼륨 조절 지원
- 온라인 TTS 실패 시 Windows SAPI 로컬 여성 음성으로 fallback
- `Windows Local` 엔진을 직접 선택할 수도 있음

### 오디오 라우팅

- 출력 대상 장치 선택
- `세팅` 버튼:
  - 앱 출력 장치를 가능한 경우 `CABLE Input`으로 변경
  - Windows 기본 캡처 장치 3종 역할(console, multimedia, communications)을 `CABLE Output`으로 변경
  - 변경 전 기본 마이크 장치 ID를 백업
- `해제` 버튼:
  - 백업된 Windows 기본 마이크 장치로 복원
  - 앱 출력 대상을 시스템 기본값으로 되돌림
- VB-CABLE 미설치 시 앱 시작 단계에서 안내 모달 표시
- 안내 모달에서 VB-CABLE 공식 다운로드 페이지 열기 지원

### 볼륨/모니터링

- `Discord 송출 볼륨`은 Discord로 보내는 send audio에만 적용
- `내 듣기`가 켜져 있으면 같은 소리를 기본 스피커로 별도 재생
- monitor audio는 볼륨 1로 고정되어, 송출 볼륨을 낮춰도 사용자가 듣는 소리는 줄어들지 않음
- 사용자가 Windows의 "이 장치로 듣기" 기능이나 VoiceMeeter 자체 모니터링으로 CABLE Output을 듣는 경우에는 케이블 신호 자체를 듣는 것이므로 송출 볼륨과 같이 줄어들 수 있음

### UI

- GPT/Codex 느낌의 밝고 깔끔한 레이아웃
- 왼쪽 사이드바, 상단 액션 버튼, 설정 스트립, TTS 작성 영역, 문장/사운드 카드 영역 구성
- lucide 아이콘 사용
- 주요 버튼:
  - `세팅`
  - `해제`
  - `장치 새로고침`
  - `보관함`
  - `정지`
  - `실행`
  - `문장 저장`
  - `사운드 추가`

## 4. 주요 코드 설명

### `src/main/main.js`

Electron 메인 프로세스 진입점이다.

담당:

- 앱 창 생성
- 사용자 데이터 경로 계산
- IPC 핸들러 등록
- 문장/사운드/TTS/오디오 라우팅 요청 처리
- 사운드 파일을 앱 보관함으로 복사
- TTS 결과 파일 URL 생성
- VB-CABLE 다운로드 페이지 열기
- 데이터 폴더 열기

중요 IPC:

- `library:get`
- `settings:update`
- `phrase:add`
- `phrase:delete`
- `sound:import`
- `sound:import-youtube`
- `sound:trim`
- `sound:delete`
- `youtube:resolve`
- `log:add`
- `tts:synthesize`
- `audio:setup-cable`
- `audio:get-cable-status`
- `audio:release-cable`
- `app:open-data-folder`
- `app:open-vb-cable-download`

### `src/main/store.js`

JSON 기반 로컬 저장소 래퍼다.

담당:

- `library.json` 생성/읽기/쓰기
- 손상되었거나 누락된 데이터는 기본값으로 정규화
- 임시 파일에 쓴 뒤 rename하여 저장 안정성 확보

저장 위치는 Electron `app.getPath("userData")` 아래다.

주요 데이터:

- 설정값
- 저장 문장 목록
- 사운드보드 목록
- TTS 전송 로그
- 이전 Windows 기본 캡처 장치 백업

### `src/main/tts.js`

TTS 합성 담당 모듈이다.

담당:

- Edge Neural TTS WebSocket 요청 생성
- SSML 생성
- MP3 오디오 프레임 수신 및 파일 저장
- Windows SAPI fallback WAV 생성
- 오래된 TTS 캐시 파일 정리

중요한 점:

- Edge Neural TTS는 온라인 연결이 필요하다.
- `TRUSTED_CLIENT_TOKEN`, `Sec-MS-GEC` 생성 로직을 사용한다.
- TTS 파일은 `tts-cache` 폴더에 저장된다.
- 기본 캐시 보존 개수는 50개다.

### `src/main/windowsAudio.js`

Windows 기본 캡처 장치 변경 담당 모듈이다.

담당:

- PowerShell에서 C# COM interop 코드를 실행
- Windows 캡처 장치 목록 조회
- 기본 캡처 장치 역할별 ID 조회
- `CABLE Output` 장치 탐색
- 기본 캡처 장치를 `CABLE Output`으로 변경
- 저장된 백업값으로 원복

변경하는 역할:

- console
- multimedia
- communications

주의:

- Windows 전용 기능이다.
- 내부적으로 `powershell.exe`를 실행한다.
- 관리자 권한 없이도 동작하도록 설계했지만, Windows 환경/권한/보안 정책에 따라 실패할 수 있다.

### `src/main/youtube.js`

YouTube 링크를 앱에서 재생 가능한 로컬 오디오 파일로 준비하는 모듈이다.

담당:

- YouTube URL 형식 검증
- 앱 데이터 폴더의 `bin/yt-dlp.exe` 존재 여부 확인
- `yt-dlp.exe`가 없으면 GitHub 최신 릴리스에서 자동 다운로드
- `yt-dlp`로 `bestaudio/best` 포맷을 다운로드
- 번들된 `ffmpeg-static`의 `ffmpeg.exe`로 MP3 추출/변환
- 최종 MP3 파일을 `youtube-cache`에 저장
- 다운로드된 로컬 파일 경로를 기존 audio 재생 파이프라인으로 전달
- 오래된 YouTube 캐시 파일 정리

주의:

- YouTube 내부 동작 변경에 영향을 받을 수 있다.
- 첫 YouTube 재생 시 `yt-dlp.exe` 다운로드 때문에 시간이 더 걸릴 수 있다.
- 앱은 YouTube 오디오를 MP3 캐시로 저장하지만, 오래된 파일은 자동 정리한다.

### `src/preload.cjs`

렌더러에서 사용할 안전한 API를 `window.voiceboard`로 노출한다.

렌더러는 Node.js에 직접 접근하지 않고 이 API를 통해 메인 프로세스와 통신한다.

## 8. 사용자에게 안내한 설치 흐름

친구에게 가장 쉬운 방식:

1. GitHub Releases에 `Voiceboard-Setup-0.1.0-x64.exe`를 업로드한다.
2. 친구는 저장소 링크로 들어간다.
3. 오른쪽 `Releases`에서 최신 버전을 누른다.
4. `Assets`에서 `Voiceboard-Setup-0.1.0-x64.exe`를 다운로드한다.
5. 실행한다.
6. SmartScreen 경고가 나오면 출처 확인 후 `추가 정보` > `실행`을 누른다.
7. VB-CABLE이 없으면 앱 안내창에서 공식 페이지로 이동해 설치한다.
8. Discord 입력 장치는 `Default`로 둔다.
9. 앱에서 `세팅`을 누르고 사용한다.
10. 끝나면 `해제`를 누른다.

주의:

- GitHub의 초록색 `Code` > `Download ZIP`은 설치 파일이 아니라 소스 코드다.
- 초보 사용자에게는 Release의 `.exe` 다운로드를 안내해야 한다.

## 9. 알려진 주의점과 한계

- Windows 전용 앱이다.
- VB-CABLE 또는 유사한 가상 오디오 케이블이 필요하다.
- Edge Neural TTS는 온라인 연결이 필요하다.
- Edge TTS 내부 프로토콜은 비공식 사용에 가깝기 때문에 장기적으로 동작이 깨질 수 있다.
- Windows 기본 오디오 장치 변경은 환경에 따라 실패할 수 있다.
- YouTube 재생은 YouTube 정책/페이지 구조 변경에 따라 일시적으로 실패할 수 있다.
- 앱이 Discord 앱 내부 설정을 직접 조작하는 것은 아니다. Discord 입력 장치를 `Default`로 두어 Windows 기본 캡처 장치 변경을 따라가게 하는 방식이다.
- 설치 파일은 코드 서명이 없으므로 SmartScreen 경고가 뜰 수 있다.
- `dist` 폴더는 Git에 올라가지 않으므로 배포하려면 GitHub Release 또는 별도 공유가 필요하다.

- 앱 자동 업데이트 또는 Release 링크 열기 추가
- UI 모바일 대응은 불필요하지만 작은 창에서의 최소 레이아웃 점검
